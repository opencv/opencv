{# Override of pydata_sphinx_theme's components/navbar-nav.html. PyData's default renders both top-level toctree items AND the configured external_links in one bar. For OpenCV we want ONLY the Doxygen-style nav (Main Page, Related Pages, Namespaces, …) in the header — the tutorial module tree belongs in the left sidebar, not the header. So we render external_links only. These are NOT external links: conf.py declares them via the theme's `external_links` key only because that's the data slot for a custom header nav, but every target resolves WITHIN this build. conf.py builds each URL from DOXYGEN_BASE_URL (default https://docs.opencv.org/5.x/) + a filename; we rewrite any URL with that prefix to a depth-aware relative path into the local Doxygen output at /doc/doxygen/html/, so the link stays on-site and resolves to the Doxygen pages built alongside this Sphinx tree. "Main Page" instead routes to the Sphinx master_doc (tutorials/tutorials) via `pathto`, because the Sphinx-rendered site IS the redesigned landing page — sending the user to the legacy Doxygen index would feel like a regression. All entries render as plain in-tab nav-links (no target="_blank", no nav-external class, no ↗ glyph) since navigation never leaves the site. The `+ 2` in _doxygen_local covers climbing out of html/ and docs_sphinx/ to reach build/doc/doxygen/html/. #} {%- set _depth = (pagename or '').count('/') %} {%- set _doxygen_local = ('../' * (_depth + 2)) ~ 'doc/doxygen/html/' %}
