🛠️ INSTALLATION QUICK START
═══════════════════════════════════════════════════════════════════════════

Prerequisites:
  1. ROCm 7.0.2 (critical version)
  2. HIP Threads library
  3. CMake 3.21+

Steps:
  # Install ROCm 7.0.2
  sudo apt update && sudo apt install -y rocm-hip-runtime-7.0.2
  
  # Install HIP Threads
  git clone https://github.com/ROCm/hipThreads.git
  cd hipThreads && cmake -B build && cmake --install build
  
  # Build OpenCV with HIP support
  cd opencv && mkdir build && cd build
  cmake .. -DWITH_HIP=ON
  make -j$(nproc)

For detailed instructions: See modules/hip/INSTALLATION.md


📖 API USAGE EXAMPLES
═══════════════════════════════════════════════════════════════════════════

Basic GPU Acceleration (transparent fallback):
  
  #include <opencv2/hip/hip_kernels.hpp>
  
  cv::Mat image = cv::imread("photo.jpg");
  cv::Mat result;
  
  // GPU if beneficial, CPU otherwise (automatic!)
  cv::hip::gaussianBlur_gpu(image, result, cv::Size(5, 5), 1.0);

Configuration:

  #include <opencv2/hip/hip_config.hpp>
  
  // Check GPU availability
  if (cv::hip::isGPUAvailable()) {
      std::cout << "GPU ready for acceleration\n";
  }
  
  // Modify settings for your use case
  auto& config = cv::hip::getGPUConfig();
  config.min_image_size_bytes = 512 * 1024;  // 512KB threshold
  config.min_flops_per_element = 5.0f;        // More aggressive

Force CPU-Only Mode:

  cv::hip::setGPUEnabled(false);  // Disable GPU entirely
